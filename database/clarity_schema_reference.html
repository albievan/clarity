SQL Server vs MariaDB — Dialect Differences
Key syntax differences between the two schema files
ConceptSQL ServerMariaDBNotes
Auto-incrementINT NOT NULL IDENTITY(1,1)INT NOT NULL AUTO_INCREMENTBoth always start at 1, increment by 1
String type (Unicode)NVARCHAR(n) — UTF-16VARCHAR(n) CHARACTER SET utf8mb4MariaDB: set at table level (CHARSET=utf8mb4)
Long textNVARCHAR(MAX)TEXT / LONGTEXTMariaDB TEXT = ~65KB; LONGTEXT = ~4GB
BooleanBIT (0 or 1)TINYINT(1)MariaDB has no native BOOL — TINYINT(1) is standard convention
Timestamp with precisionDATETIME2(3)DATETIME or DATETIME(3)MariaDB DATETIME(3) = millisecond precision; audit_log uses DATETIME(3)
Current UTC timestamp defaultDEFAULT GETUTCDATE()DEFAULT CURRENT_TIMESTAMPMariaDB CURRENT_TIMESTAMP uses server timezone — ensure tz=UTC in my.cnf
Auto-update timestampRequires trigger or app logicON UPDATE CURRENT_TIMESTAMPMariaDB supports this natively on DATETIME columns
JSON columnNVARCHAR(MAX) — no native JSON type, use JSON functionsJSON (alias for LONGTEXT with JSON validation)audit_log.before_state / after_state use JSON in MariaDB
Schema namespacingplatform.tenants, auth.users, finance.budgets etc.Single flat namespace (no schemas) — use one DB per tenant or prefix tablesSQL Server schemas aid logical grouping and permissions; MariaDB uses database-level isolation
Multi-tenant isolationSchema-per-schema logical grouping; row-level tenant_id filterDatabase-per-tenant (recommended) or row-level tenant_id filterBoth schemas implement row-level tenant_id on every table for flexibility
CHECK constraintsFully enforced since SQL Server 2019Parsed but not enforced before MariaDB 10.2.1; enforced 10.2.1+Verify MariaDB version supports CHECK enforcement
Filtered / partial indexCREATE INDEX ... WHERE is_contract_linked = 1Not supported — use full index or generated columnMariaDB: CREATE INDEX ix ON budget_lines (is_contract_linked) WHERE clause not supported
Default DB engineN/A — SQL Server onlyENGINE=InnoDB required — MyISAM does not support FKAll MariaDB tables explicitly set ENGINE=InnoDB
Self-referencing FK (departments)Supported inlineSupported inlineIdentical — departments.parent_dept_id → departments.id
Forward reference FK (fx_rates → budget_periods)Handle with ALTER TABLE or correct creation orderSET foreign_key_checks=0 at top, SET foreign_key_checks=1 at bottom; then ALTER TABLE fx_rates ADD CONSTRAINT...MariaDB file uses deferred FK addition for circular/forward references

Running Balance Formula
How Clarity computes remaining budget in vw_budget_line_running_balance
-- Running balance per budget line
Remaining = total_budget
           − SUM(purchase_orders.total_value WHERE status != 'closed') ← Committed
           − SUM(actuals.amount WHERE is_deleted = 0) ← Confirmed actuals

-- When goods receipt converts committed → actuals:
-- PO encumbrance reduces (or PO closes), actuals increases by same amount.
-- Net effect on Remaining = zero. Only un-receipted PO value counts as committed.

-- RAG thresholds (from BRD FR-064 / AR-001)
Green = pct_committed_or_spent <= 95%
Amber = pct_committed_or_spent 95% – 105%
Red = pct_committed_or_spent > 105%
